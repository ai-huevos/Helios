# Writing Review Skill

Review and improve written content for clarity, correctness, and impact.

## Quick Start

| Review Type | Focus |
|-------------|-------|
| **Quick Edit** | Grammar, typos, basic clarity |
| **Structural Review** | Organization, flow, completeness |
| **Tone Calibration** | Audience fit, voice consistency |
| **Deep Edit** | Full rewrite for maximum impact |

## Review Framework

### 1. Read First (Don't Edit Yet)
- What's the purpose?
- Who's the audience?
- What's the desired outcome?

### 2. Flag Issues (Categories)

### 3. Suggest Changes
- Provide specific rewrites
- Explain the "why" briefly
- Preserve author's voice

## Audience-Specific Style

| Audience | Style | Avoid |
|----------|-------|-------|
| **Investors** | Confident, data-driven, concise | Hedging, fluff |
| **Clients** | Professional, clear, helpful | Jargon, condescension |
| **Technical** | Precise, complete, structured | Vagueness |
| **Marketing** | Energetic, benefit-focused | Features-only |
| **Internal** | Direct, casual, efficient | Formality |

## Quality Levels

### Level 1: Clean (Minimum)
- No spelling/grammar errors
- Complete sentences
- Consistent formatting

### Level 2: Clear (Standard)
- Easy to understand
- Logical flow
- Appropriate tone

### Level 3: Compelling (Premium)
- Engaging opening
- Strong transitions
- Memorable closing
- Quotable phrases

## Review Workflow

```
1. PURPOSE: Why was this written?
2. AUDIENCE: Who will read it?
3. SCAN: Get overall impression
4. STRUCTURE: Is organization logical?
5. CLARITY: Is each point clear?
6. GRAMMAR: Are mechanics correct?
7. TONE: Does voice match purpose?
8. FINAL: Read aloud for flow
```

## When NOT to Edit

- Don't change author's voice unnecessarily
- Don't over-formalize casual content
- Don't add your opinions as "corrections"
- Don't rewrite when light editing suffices
- Preserve intentional style choices
